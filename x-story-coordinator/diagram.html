üîç Phase 4: Story Quality Delegation (Orchestrator-Worker Pattern)

  • Pass 1 Invocation: x-story-coordinator delegates to x-story-quality-coordinator Pass 1 (Code Quality ‚Üí Regression ‚Üí Manual Testing with Early Exit)
  • Auto-Verification: New tasks (test/fix/refactor) created in Backlog are automatically verified by x-story-validator (invoked via Skill tool)
  • Loop Back: After auto-verification moves tasks to Todo, orchestration loop continues via Priority 2 (Todo execution)
  • Pass 2 Invocation: When test task Done, Story status updates to To Review and x-story-quality-coordinator Pass 2 is invoked automatically
  • Story Done: Pass 2 verifies tests (Priority ‚â•15, 10-28 total), updates Story status to Done, workflow complete
  • Self-Healing: Pass 2 failures create fix tasks in Backlog ‚Üí x-story-validator auto-verification ‚Üí loop back to Phase 3
